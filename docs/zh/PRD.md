# GhostLink (幽灵链路) — 产品需求文档

## 1. 产品概述

### 1.1 产品名称

GhostLink（幽灵链路）

### 1.2 产品定位

基于 Solana 区块链的端到端加密点对点消息通信工具。利用链上 Memo 存储和非对称加密技术，实现完全隐私的消息传递，确保只有收件方能够解密和阅读消息内容。

### 1.3 核心价值

- **隐私性**：消息使用收件方公钥加密，只有持有对应私钥的收件方能解密
- **匿名性**：通过隐形收件箱和 Tor 代理，隐藏通信双方身份
- **去中心化**：消息存储在 Solana 链上，无中心化服务器，不可篡改
- **抗审查**：基于区块链的存储方案天然抗审查

## 2. 目标用户

- 对隐私有高度需求的个人用户
- 需要安全通信渠道的组织
- 加密货币社区用户
- 隐私倡导者和安全研究人员

## 3. 技术规格

### 3.1 开发语言

Go (Golang)

### 3.2 产品形态

命令行工具（CLI）

### 3.3 区块链网络

Solana devnet

### 3.4 存储方案

Solana Memo Program，单条消息上限 **512 字节**

### 3.5 加密方案

非对称加密（基于 Solana 密钥对）：

- 发送方使用收件方的 **公钥** 加密消息
- 收件方使用自己的 **私钥** 解密消息

### 3.6 内容类型

仅支持 **纯文本** 消息

## 4. 功能需求

### 4.1 CLI 子命令结构

| 子命令 | 功能 | 优先级 |
|--------|------|--------|
| `send` | 发送加密消息 | P0 |
| `receive` | 接收并解密消息 | P0 |
| `inbox create` | 创建隐形收件箱 | P0 |
| `inbox share` | 生成收件箱二维码 | P1 |
| `wallet create` | 创建/导入 Solana 钱包 | P0 |
| `wallet balance` | 查询钱包余额 | P1 |

### 4.2 发送消息（send）

**描述**：向指定收件地址发送加密文本消息。

**输入参数**：

| 参数 | 说明 | 必填 |
|------|------|------|
| `--to` | 收件方地址（公钥或收件箱地址） | 是 |
| `--message` / `-m` | 消息内容（纯文本） | 是 |
| `--tor` | 是否通过 Tor 代理发送 | 否 |

**处理流程**：

1. 验证消息内容加密后不超过 512 字节
2. 获取收件方公钥
3. 使用收件方公钥加密消息内容
4. 构造 Solana 交易，将加密内容写入 Memo 指令
5. 签名并提交交易到 Solana 网络
6. 返回交易签名（tx hash）作为发送凭证

**错误处理**：

- 消息超过 512 字节限制时提示用户缩短内容
- 余额不足时提示充值
- 网络异常时重试或提示

### 4.3 接收消息（receive）

**描述**：扫描链上交易，解密并展示收到的消息。

**输入参数**：

| 参数 | 说明 | 必填 |
|------|------|------|
| `--inbox` | 指定收件箱地址 | 否 |
| `--limit` | 最大返回消息数 | 否 |
| `--since` | 起始时间过滤 | 否 |

**处理流程**：

1. 查询指定地址（或默认钱包地址）的相关交易
2. 过滤包含 Memo 指令的交易
3. 使用用户私钥尝试解密 Memo 内容
4. 格式化展示解密后的消息（发送方、时间、内容）

### 4.4 隐形收件箱（inbox）

**描述**：利用 Solana Address Lookup Tables (ALT/LUT) 创建隐形收件箱地址。

**核心设计**：

- 收件箱地址由用户本地生成，私密不公开
- 使用 Address Lookup Table 映射真实地址，增加追踪难度
- 收件箱地址可通过二维码安全分享给发送方
- 一个用户可以创建多个收件箱，用于不同场景

**二维码分享**：

- 生成包含收件箱地址信息的二维码
- 二维码可输出为终端字符画或 PNG 图片
- 扫码方可直接获取收件箱地址用于发送消息

### 4.5 Tor 代理集成

**描述**：通过 Tor 网络代理所有与 Solana RPC 节点的通信。

**功能要求**：

- 支持通过 `--tor` 标志启用 Tor 代理
- 支持配置文件中设置默认启用 Tor
- 支持自定义 SOCKS5 代理地址（默认 `127.0.0.1:9050`）
- 隐藏用户 IP 地址，防止 RPC 节点关联用户身份

### 4.6 钱包管理（wallet）

**描述**：管理 Solana 钱包密钥对。

**功能要求**：

- 创建新的 Solana 密钥对
- 导入已有私钥（支持助记词 / Base58 私钥）
- 本地加密存储私钥（密码保护）
- 查询钱包 SOL 余额

## 5. 非功能需求

### 5.1 安全性

| 要求 | 说明 |
|------|------|
| 端到端加密 | 消息从发送到接收全程加密，中间不存在明文 |
| 私钥本地存储 | 私钥仅保存在用户本地，加密存储 |
| 内存安全 | 敏感数据用完即清，不残留在内存中 |
| 零知识 | 系统不保存任何元数据，不追踪用户行为 |

### 5.2 性能

| 指标 | 目标值 |
|------|--------|
| 消息发送延迟 | < 5 秒（不含 Tor） |
| 消息接收延迟 | < 3 秒（获取最新消息） |
| 加解密耗时 | < 100ms |

### 5.3 可用性

- 支持 Linux / macOS / Windows 三平台交叉编译
- 单二进制文件分发，无外部依赖
- 提供清晰的命令行帮助文档（`--help`）
- 有意义的错误提示信息

## 6. 约束与限制

| 约束 | 说明 |
|------|------|
| 消息大小 | 加密后不超过 512 字节（Memo 限制） |
| 内容类型 | 仅支持纯文本，不支持文件/图片/语音 |
| 费用 | 每条消息需消耗少量 SOL 作为交易费 |
| 网络依赖 | 需要连接 Solana 网络（可通过 Tor） |
| 消息不可撤回 | 链上数据不可删除或修改 |

## 8. 术语表

| 术语 | 说明 |
|------|------|
| **Memo** | Solana 链上 Memo Program，允许在交易中附加最多 512 字节的数据 |
| **ALT / LUT** | Address Lookup Table，Solana 的地址查找表，用于压缩交易中的地址引用 |
| **Tor** | 洋葱路由网络，通过多层加密和中继节点隐藏用户真实 IP |
| **SOCKS5** | 网络代理协议，Tor 默认使用 SOCKS5 代理 |
| **端到端加密** | 消息仅在发送方和接收方之间可读，中间节点无法解密 |
